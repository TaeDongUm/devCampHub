# 우아한테크코스 미션: 네트워크 장애 상황 시뮬레이션 및 개선

## 0. 미션 배경 및 동기

1. **3주간의 프리코스를 통해 배운 것**

```text
지난 3주간의 우아한테크코스 프리코스를 통해, 테스트 주도 개발(TDD) 방법론과 객체지향을 학습하며 견고하고 유지보수하기 좋은 코드를 작성하는 역량을 길렀습니다. 이는 주로 애플리케이션의 내부 안정성을 높이는 데 중점을 둔 과정이었습니다.
```

2. **"네트워크 장애 시뮬레이션" 목표 설정 이유**

```text
프리코스에서 배운 역량을 한 단계 확장하고 싶었습니다. 안정적인 소프트웨어는 내부 설계뿐만 아니라, 예측 불가능한 외부 환경의 문제에 어떻게 대응하는가에 따라 완성된다고 생각합니다.

특히 실시간 통신이 핵심인 devCampHub 프로젝트에서 '네트워크'라는 외부 환경의 불확실성에 대처하는 능력을 기르는 것이, 프리코스에서 배운 내부 안정성 확보 역량을 확장시킬 수 있는 도전이라고 판단하여 이 미션 목표를 설정하게 되었습니다.
```

3. **팀 단위가 아닌 개인으로 진행한 이유**

```text
이번 오픈 미션을 팀 단위로 수행할 경우, 팀이 합의한 방향과 우선순위에 따라 미션의 결이 정해질 것이라 판단했습니다. 팀 미션은 자연스럽게 의사소통, 조율, 합의에 많은 시간과 에너지가 투입되고, 이는 분명 협업 역량 측면에서 큰 도전이자 가치 있는 성장입니다.

다만 저는 이번 미션에서, 우테코 프리코스를 통해 익힌 내부 안정성 중심의 학습 루프를 외부 환경(네트워크)으로 확장해 보고자 했습니다. 개인 진행은 합의 지연 없이 가설 설정 -> 시뮬레이션 -> 측정 -> 개선 -> 재측정의 사이클을 짧게 돌릴 수 있어, 같은 시간에 더 많은 실험을 수행하기에 유리하다고 판단했습니다.
```

3. **이 미션이 나에게 '도전'인 이유**

```text
devCampHub 프로젝트는 저의 첫 번째 WebSocket 기반 실시간 통신 프로젝트입니다. GitHub 저장소에서 확인할 수 있듯, 기능 구현 자체에 집중해왔기에 네트워크 지연, 끊김, 패킷 손실 등 다양한 장애 상황에 대한 깊이 있는 학습이나 대비는 전무한 상태입니다.

따라서 저에게 익숙하지 않은 네트워크의 동작 원리를 학습하고, 장애 상황을 직접 시뮬레이션하며 이를 코드 레벨에서 개선해 나가는 과정 전체가 저에게는 의미 있는 '성장통'이자 '도전'입니다.
```

## 1. 미션 목표

본 미션의 목표는 현재 진행 중인 `devCampHub` 프로젝트에 의도적으로 네트워크 장애 상황을 시뮬레이션하고, 이를 분석 및 개선하는 과정을 통해 네트워크에 대한 이해를 높이고 문제 해결 능력을 기르는 것입니다.

## 2. 미션 진행 계획

1.  **프로젝트 현황 분석 및 정리 (As-Is)**

    - `devCampHub` 프로젝트의 현재 기술 스택, 아키텍처, 핵심 기능 등을 분석하고 문서화합니다.

2.  **네트워크 장애 상황 학습 및 선정**

    - 웹 애플리케이션에서 발생할 수 있는 다양한 네트워크 장애(지연, 패킷 손실, 대역폭 제한 등)에 대해 학습합니다.
    - `devCampHub` 프로젝트의 특성(실시간 스트리밍, 채팅)에 가장 큰 영향을 줄 수 있는 장애 시나리오를 1~2개 선정합니다.

3.  **장애 상황 시뮬레이션 및 분석**

    - 선정한 장애 상황을 프로젝트에 의도적으로 발생시킵니다.
    - Wireshark, Charles Proxy 등 네트워크 분석 도구를 학습하고 선택합니다.
    - 선택한 도구를 사용하여 장애 상황을 측정하고, 데이터를 통해 현상을 명확하게 분석합니다.

4.  **개선 방안 적용 (To-Be)**

    - 분석한 내용을 바탕으로, 장애 상황을 완화하거나 사용자 경험을 개선할 수 있는 코드를 작성하고 적용합니다.

5.  **개선 결과 검증 및 비교**

    - 동일한 네트워크 분석 도구를 사용하여 개선된 버전의 애플리케이션을 다시 측정합니다.
    - 개선 전/후의 데이터를 비교하여 개선 효과를 정량적으로 분석하고 문서화합니다.

6.  **회고**
    - 미션 전체 과정을 통해 배우고 느낀 점, 어려웠던 점 등을 정리하며 회고합니다.

## 3. 미션 시작점

이 `MISSION.md` 파일의 첫 번째 커밋을 미션의 시작점으로 하며, `mission/network-sim/start` Git 태그를 함께 사용합니다.
