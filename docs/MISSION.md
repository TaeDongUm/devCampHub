# 우아한테크코스 미션: 네트워크 장애 상황 시뮬레이션 및 개선

## 1. 미션 목표

본 미션의 목표는 현재 진행 중인 `devCampHub` 프로젝트에 의도적으로 네트워크 장애 상황을 시뮬레이션하고, 이를 분석 및 개선하는 과정을 통해 네트워크에 대한 이해를 높이고 문제 해결 능력을 기르는 것입니다.

## 2. 미션 진행 계획

1.  **프로젝트 현황 분석 및 정리 (As-Is)**

    - `devCampHub` 프로젝트의 현재 기술 스택, 아키텍처, 핵심 기능 등을 분석하고 문서화합니다.

2.  **네트워크 장애 상황 학습 및 선정**

    - 웹 애플리케이션에서 발생할 수 있는 다양한 네트워크 장애(지연, 패킷 손실, 대역폭 제한 등)에 대해 학습합니다.
    - `devCampHub` 프로젝트의 특성(실시간 스트리밍, 채팅)에 가장 큰 영향을 줄 수 있는 장애 시나리오를 1~2개 선정합니다.

3.  **장애 상황 시뮬레이션 및 분석**

    - 선정한 장애 상황을 프로젝트에 의도적으로 발생시킵니다.
    - Wireshark, Charles Proxy 등 네트워크 분석 도구를 학습하고 선택합니다.
    - 선택한 도구를 사용하여 장애 상황을 측정하고, 데이터를 통해 현상을 명확하게 분석합니다.

4.  **개선 방안 적용 (To-Be)**

    - 분석한 내용을 바탕으로, 장애 상황을 완화하거나 사용자 경험을 개선할 수 있는 코드를 작성하고 적용합니다.

5.  **개선 결과 검증 및 비교**

    - 동일한 네트워크 분석 도구를 사용하여 개선된 버전의 애플리케이션을 다시 측정합니다.
    - 개선 전/후의 데이터를 비교하여 개선 효과를 정량적으로 분석하고 문서화합니다.

6.  **회고**
    - 미션 전체 과정을 통해 배우고 느낀 점, 어려웠던 점 등을 정리하며 회고합니다.

## 3. 미션 시작점

이 `MISSION.md` 파일의 첫 번째 커밋을 미션의 시작점으로 하며, `mission/network-sim/start` Git 태그를 함께 사용합니다.
